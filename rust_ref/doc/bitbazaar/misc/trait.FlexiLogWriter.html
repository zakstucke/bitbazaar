Expand description

The actual object used to apply logs. This is useful when updates are async, making synchronous batching easy.

Required Methods§

source

Get the current phase. Just always return e,g, Pending if not supported.

fn set_progress(&self, progress: f64)

Set the current progress. Depending on the underlying imp this could e.g. log a message, or write a progress bar etc.

source

Set the current phase. The underlying imp might not do anything for phases.

source

fn log_with_opts(&self, lvl: Level, msg: String, force_replace_prior: bool)

Underlying log adder with more options. If the actual output doesn’t support force_replace_prior, just ignore it and add like normal. (Same with lvl.)
